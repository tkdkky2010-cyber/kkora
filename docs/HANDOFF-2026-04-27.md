## 1. 이 대화에서 결정/확정된 것

### 결제 / 자금 보관 구조 (가장 중요)

**최종 방향**: 카카오페이머니 네이티브 모델 채택 (단, PG 미팅에서 확인 필요)

```
[충전] 사용자 카카오페이 결제 → 카카오페이 에스크로 (꺼라 가맹점 명의)
[잔액] 우리 Firestore DB에 사용자별 잔액 기록 (또는 카카오페이 sub-account)
[챌린지] DB 차감만 (실제 자금 이동 없음)
[정산] 카카오페이 송금 API로 결과 분배
[출금] 사용자가 카카오페이 앱에서 본인 계좌로 (우리 펌뱅킹 불필요)
```

**이유**:
- 자금이 우리 통장에 안 들어옴 → 전자금융거래법 등록 의무 면제
- 카카오페이 모든 라이선스 보유 (PG/에스크로/송금/선불전자지급수단)
- 수수료 0.7~1.5% (영세 등급 시)
- 등록 의무 면제 + 펌뱅킹 인프라 불필요

**백업**: 토스페이먼츠 PG (카카오페이 안 쓰는 30% 사용자용)

### 비즈니스 모델 수정사항 (CLAUDE.md 업데이트 대기)

| 항목 | 기존 (CLAUDE.md) | 변경 |
|------|--------------|------|
| 충전 단위 | 1,000 / 5,000 / 10,000원 | **3,000 / 5,000 / 10,000원** (1,000원 제거) |
| 플랫폼 수수료 | 미환급액의 20% | **미환급액의 30%** (또는 챌린저스 룰 검토 중) |
| 출금 최소액 | 3,000원 | **5,000원** |
| 출금 수수료 | 명시 없음 | **월 1회 무료, 추가 시 500원 고정** |
| 금요일 0% 수수료 | 있음 | **폐지 검토 중** (적자 위험 발견됨) |
| 충전 수수료 사용자 부담 | — | **사용자 부담 X (플랫폼 흡수)** |

### 검토 중 (확정 안 된 것)
- **수수료 모델**: 현행 30% vs 챌린저스 룰 (총예치금 5% OR 미환급 50% 중 큰 값) — 미결
- **금요일 0%**: 완전 폐지 vs 월 1회로 축소 — 미결
- **결제 수단 최종**: 카카오페이 + 토스페이 + 퀵계좌이체 — PG 미팅 후 확정

## 2. 검증된 핵심 사실 (검색으로 확인됨)

### PG 수수료 (정확 수치)

| PG | 카드 | 계좌이체 | 카카오페이 직결제 |
|----|----|------|---------------|
| 토스페이먼츠 PG | 3.4% (VAT 별도) | 2.0% (퀵계좌이체) | — |
| 카카오페이 직접 | — | — | **0.56~1.72% (카드)** / **0.72~1.55% (머니)** |

**기타 비용**:
- 토스페이먼츠 가입비 220,000원 + 연관리비 110,000원
- 모든 수수료에 VAT 10% 별도
- 토스페이먼츠 에스크로 추가: 카드 0.2% / 계좌이체 0.2% / 가상계좌 200원
- 토스페이먼츠 계좌 유효성 인증: 건당 50원

### 영세/중소 가맹점 분류 (확인됨)
- **영세**: 연매출 3억원 이하
- **중소**: 연매출 3억원 초과 ~ 30억원 이하
- **일반**: 연매출 30억원 초과
- 분류 단위: **사업자등록번호 단위** (개인 명의 단위 아님)
- **신규 사업자 = 처음 6개월 일반 등급** → 6개월 후 영세 분류 + 소급 환급
- 다른 사업체 매출(30억+)과 별개 판정 → 새 사업자는 정상적으로 영세 가능

### 챌린저스 (벤치마크)
- 수수료: **총예치금의 5% OR 미환급액의 50% 중 큰 값**
- 85%+ 성공률 시 예치금 전액 환급
- 출처: 챌린저스 이용약관 (https://biz-challengers.com/terms)
- 평균 실패율 약 20% (업계 추정)

### 법적 환경
- 전자금융업 자체 등록: **자본금 20억원 + 인력/시스템** → 스타트업 사실상 불가
- 미등록 시: **5년 이하 징역 또는 3천만원 이하 벌금**
- **PG/에스크로 위탁 구조 = 등록 의무 회피의 표준 방법**
- 챌린저스 7년간 운영, 형사처벌 사례 없음 → 합법 모델로 검증됨

## 3. Sub-account 개념 (꼭 알아야 할 핵심)

**Sub-account**: PG 시스템 안에서 사용자별로 잔액을 분리 추적

### 두 가지 방식 비교

#### 방식 A: 카카오페이가 사용자별 잔액 직접 추적 (Sub-account 지원)
```
카카오페이 시스템 내부:
  꺼라 에스크로 (총 5,000만원)
   ├─ userA: 50,000원
   ├─ userB: 30,000원
   └─ ... (사용자별 분리 추적)
```
- 우리 도산해도 사용자가 카카오페이에 직접 청구 가능
- ✅ 등록 의무 면제 명확

#### 방식 B: 카카오페이는 총합만, 우리 DB가 사용자별 추적
```
카카오페이: 꺼라 에스크로 5,000만원 (총합만)
우리 Firestore: users/userA.balance = 50,000원 (사용자별 우리만 알고 있음)
```
- 우리 DB가 망가지면 사용자 권리 입증 어려움
- ⚠️ 법적 회색지대 ("실질적 자금 관리자 = 등록 의무자" 판정 위험)

**카카오페이 표준 가맹점은 보통 sub-account 미지원** → 별도 협의 필요

→ 미팅에서 가장 중요한 질문: "꺼라처럼 사용자별 예치금 분리 추적 가능?"

## 7. 발견된 함정 / 정정 사항

대화 중 잘못된 분석을 정정한 부분 (다음 컴퓨터 Claude도 알아야 함):

### 정정 1: "수수료 0% = 그날 손익 영향 미미"는 틀림
- PG 충전 수수료(3%)는 플랫폼 수수료와 무관하게 무조건 발생
- 1,000명 × 만원 = 천만원 풀의 0% 이벤트 → 충전 수수료 30만원 그대로 손실
- 참여자 많을수록 손실 비례 증가
- → **금요일 0% 폐지 또는 축소 필요**

### 정정 2: "카카오페이는 약 3% 수수료"는 부정확
- 정확한 수치: **0.56~1.72% (카드)** / **0.72~1.55% (머니)**
- 영세 등급 시 0.6% 수준 = 거의 비용 없음
- 토스페이먼츠 PG 카드 결제 3.4%와 혼동했음

### 정정 3: "PG 위탁 구조 = 카카오페이 결제 자동 포함"은 오해
- 결제 ≠ 자금 보관
- 카카오페이로 받아도 우리 통장에 입금되면 등록 의무 발생
- 카카오페이 에스크로에 들어가야 면제

### 정정 4: 카카오페이도 PG임 (이전 설명 헷갈림)
- 카카오페이는 PG/에스크로/송금/선불 모든 라이선스 보유
- 별도 PG 끼울 필요 없을 수 있음
- 카카오페이 직접 + 단독 운영 가능성 있음
